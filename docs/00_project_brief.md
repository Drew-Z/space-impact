# 项目简介

## 项目目标

本项目使用 Godot 4.6.1 复刻 / 重制 Nokia 3310 上的《Space Impact》初代体验，优先还原其“简洁、直接、短局、街机感”的核心，而不是扩展成现代原创横版射击游戏。

## 目标玩家体验

- 进入游戏后能快速理解操作与目标。
- 在短时间内体验持续推进的横向卷轴战斗。
- 通过基础射击、躲避、拾取强化和 Boss 战获得紧凑的街机节奏。
- 视觉与反馈让玩家立刻联想到 Nokia 3310 单色屏时代的《Space Impact》。

## 忠实复刻范围

- 横向自动卷轴射击的整体结构。
- 简洁清楚的敌人波次与关卡推进。
- 玩家单机单机位控制，不引入复杂成长树或关卡外养成。
- 强化拾取与关底 Boss 作为核心节奏节点。
- 单色 / 低分彩屏气质、像素感和简洁 UI。

## 允许的现代化微调

- 默认键盘映射适配现代 PC。
- 适度提升 HUD 可读性和菜单可用性。
- 为了开发和测试效率允许使用占位资源。
- 在不改变核心气质的前提下，适度调整碰撞判定、镜头节奏和反馈强度。

## 明确不做

- Roguelike build、技能树、复杂剧情、联机、多周目外层系统。
- 早期阶段加入大量特效演出、复杂弹幕或脱离原作气质的新武器体系。
- 在阶段 0 直接实现大规模玩法内容。

## 最小可玩版本定义

阶段 1 的最小可玩版本必须满足：

- 可从主菜单进入游戏。
- 有 1 个完整关卡流程。
- 玩家可移动、射击、受伤和死亡。
- 至少 2-3 类基础敌人。
- 至少 1 个强化拾取。
- 至少 1 个 Boss。
- 可进入结算并选择重开或返回菜单。

## 当前阶段结论

阶段 0 的成功标准不是“已经能玩”，而是：

- 项目结构已经初始化。
- 关键设计、技术、制作和测试文档已经有统一口径。
- 后续阶段的实现边界与验收标准已经明确。

## 参考歧义记录

当前已知存在以下歧义，后续如需更细实现时要先核对资料再下结论：

- 原版不同资料对敌人编队细节、武器名称和具体关卡数量的描述不完全一致。
- 玩家生命、受击表现和强化节奏在不同回忆资料与移植版本中可能有差异。
- 单色屏视觉在现代显示器上的还原方式存在多种可接受实现。

阶段 0 先以“忠于 2000 年 Nokia 3310 初代体感”为总目标，不擅自细化为未经核实的具体数值。
