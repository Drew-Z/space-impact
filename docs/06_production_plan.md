# 分阶段开发计划

## 阶段 0：预制作与文档

目标：

- 初始化 Godot 项目骨架。
- 固定复刻范围、核心循环、技术结构、风格方向和测试计划。

验收标准：

- `docs/00` 到 `docs/07` 已建立初稿。
- 目录结构清晰。
- 项目可被 Godot 4.6.1 识别为有效工程。

## 阶段 1：最小可玩原型

目标：

- 做出 1 个完整可玩的 Space Impact 风格关卡闭环。

交付内容：

- 主菜单
- 玩家移动与射击
- 自动卷轴
- 2-3 类基础敌人
- 1 个强化拾取
- 1 个 Boss
- 结算页

验收标准：

- 可从菜单进游戏。
- 可完成 1 关并进入结算。
- 玩家死亡与重开流程正常。

当前实现对应：

- 已开始落地 1 个主菜单、1 个固定关卡、1 个 Boss 与 1 个结果页。
- 本阶段仍需以可运行验证为准，确认流程闭环和脚本无误。

## 阶段 2：战斗与关卡扩展

目标：

- 扩展到更接近原版的多波次 / 多关卡街机体验。

交付内容：

- 更完整的敌人编队
- 更明确的关卡节奏
- 更多强化或武器变化
- 更有差异的 Boss

验收标准：

- 关卡与战斗结构明显丰富。
- 原作气质更稳定，不偏现代弹幕。

当前实现对应：

- 从单关扩展为双 Sector 流程。
- 已加入第 4 类基础敌人和额外强化种类。
- 已加入第二种 Boss 轮廓与攻击风格。
- 仍保持局内成长简单，不引入复杂 build 系统。

## 阶段 3：UI、菜单、结算与外层流程

目标：

- 让项目具备完整展示与测试流程。

交付内容：

- 完整菜单 / 暂停 / 结算
- HUD 完善
- 分数记录或最高分
- 更统一的反馈和 UI 结构

验收标准：

- 从菜单到战斗到结算全流程无断点。
- 新玩家可以快速理解当前状态。

当前实现对应：

- 主菜单已补充最高分与最近一局摘要。
- 战斗内已补暂停面板、底部操作提示、中心提示和进度信息。
- 结算页已补最高分与破纪录提示。
- 仍需人工试玩确认暂停操作、HUD 可读性和反馈强度。

## 阶段 4：打磨、平衡、音画与发布准备

目标：

- 将项目从“可玩原型”推进到“可交付复刻项目”。

交付内容：

- 平衡与手感调优
- 音效与必要音乐
- 占位资源替换与风格统一
- 文档收尾

验收标准：

- 项目结构清晰。
- 文档齐全。
- 具备稳定演示和继续迭代条件。

当前实现对应：

- 已做最后一轮手感和难度微调。
- 已补程序化音效与必要音乐氛围。
- 已统一菜单、HUD、暂停、结算与战斗反馈风格。
- 已补齐 `docs/08`、`docs/09`、`docs/10`。
- 当前剩余缺口主要在人工试玩确认，而不是结构缺失。

## 风险控制

- 任何阶段如果出现范围膨胀，优先砍掉非核心内容，而不是延迟闭环。
- 若发现参考资料冲突，先记录到文档，再决定是否调整实现目标。

## 发布后小版本方向

当前建议的发布后迭代应控制在 `v1.1` 级别的小范围打磨，重点是：

- 更贴近原版的双 Sector 节奏
- 更清晰的敌我子弹与命中反馈
- Boss 残血压迫与击破完成感
- 更稳定的暂停 / 分发 / 导出流程

不建议在这个阶段引入复杂新系统，以免破坏当前已经完成的复刻闭环。
