# Postmortem

## 项目结果

本项目已经从空目录推进到一个可完整游玩、可展示、可继续维护的 Godot 4.6.1 复刻项目。当前版本围绕 Nokia 3310 初代《Space Impact》的核心体验完成了五个常规 Sector、一个最终 Boss、13 级武器成长、强化与掉落循环、暂停 / 结算 / 最高分 / 阶段继续、以及统一的低彩复古界面与程序化音频。

## 做得好的部分

- 阶段推进始终遵循“先文档、后实现；先闭环、后扩展；先验证、再继续”的流程，没有一开始就失控扩系统。
- 项目始终围绕“简洁、直接、短局、高压”的原作气质推进，没有偏成现代原创弹幕射击。
- 菜单、HUD、暂停、结果页、Boss 预警、最终通关承接等外层流程已经收口到较完整的展示状态。
- 武器成长线从基础散射一路推进到贯穿主炮，形成了比较明确的局内成长满足感。
- Boss 战已经从最初的“能打”扩展为“有阶段感、有收尾感、有支援循环”的完整街机考验。

## 关键取舍

- 美术没有走大量素材生产路线，而是优先用几何化视觉、低彩背景主题与程序化反馈完成风格统一。
- 音频优先保证“可区分、可支撑节奏”，因此采用程序化音效与极简旋律，而不是商业级定制资源。
- 关卡、Boss 与生成数据仍主要保留在 `game_root.gd` 中，优先换来迭代速度和集中调参效率，没有过早拆成复杂资源系统。
- `阶段继续` 当前选择了“从最高已解锁阶段重新开一局”的语义，而没有继续扩成真正的中途断点存档。

## 当前最接近原作的部分

- 横向自动卷轴与短局推进节奏
- 简洁清晰的敌人编队逻辑
- 强化拾取驱动的火力成长
- 关底 Boss 的压阵感
- 单色 / 低彩屏幕气质下的高可读性战斗画面

## 与原作仍有差异的部分

- 仍然做了少量现代化可用性优化，例如更明确的 HUD、暂停、结果页、语言切换与阶段继续入口
- 武器成长和 Boss 表现比原版更展开，适合现代展示，但也比原作更“厚”一些
- 程序化音频更偏工程化完成方案，而不是高度还原的音源模拟

## 主要剩余工作

- 最后一轮人工试玩驱动的平衡微调
- 导出版完整实机验证
- 确认 Headless 环境中的两个警告是否仅为测试退出噪声

## 如果继续迭代

- 优先继续做人工试玩记录与数值微调，而不是横向扩更多系统
- 可逐步把关卡、Boss、掉落参数抽成更清晰的数据配置
- 若后续需要更高展示质量，可替换部分程序化音效与背景表现，但要保持当前复古克制方向

## 最终结论

项目已经达到“可维护、可展示、可交付”的完成版区间。当前最大的剩余工作不再是结构补洞，而是发布前最后一轮人工确认与极小范围的平衡精修。
