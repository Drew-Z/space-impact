# Playtest Report

## 版本范围

- 版本目标：阶段 4 收口后增强版
- 引擎：Godot 4.6.1
- 测试重点：完整流程、五关 + 最终 Boss 节奏、武器成长、稳定性与反馈可读性

## 已完成测试

### 自动化 / 运行验证

- 主菜单场景可启动。
- 战斗场景可启动。
- 结算场景可启动。
- 战斗场景可在固定帧率下连续运行约 4800 帧。
- 本轮先后发现并修复了 1 个武器脚本类型问题与 2 个物理回调时序问题。
- 修复后未发现新的 GDScript 解析错误。

### 结构与流程检查

- 菜单 -> 战斗 -> 结算主流程已连通。
- 暂停层已存在继续 / 重开 / 返回主菜单入口。
- 最高分与总局数已接入持久化逻辑。
- 五关 + 最终 Boss、强化、支援敌机和 13 级武器成长保持在同一闭环内。

## 当前体验判断

### 优点

- 项目已经具备“第一次打开就能理解怎么开始、怎么打一局、怎么结束”的完整流程。
- 画面、HUD、暂停页、结果页的风格基本统一。
- 战斗节奏比阶段 1 和阶段 2 更稳，反馈更明确。
- 程序化音效让发射、受击、拾取、Boss 警报和结算都有最低限度的听觉区分。

### 人工试玩反馈（2026-04-11）

- 首次人工试玩发现主菜单面板曾错误偏到右下角，导致初次进入流程受阻；该问题已修正为真正的居中布局。
- 玩家反馈原先飞船前进空间过小，只能停留在左侧约三分之一区域；当前已放宽为接近全屏安全边界，并以顶部 / 底部 HUD 作为垂直活动边界。
- 玩家反馈武器升级更适合由击毁敌机数量触发；当前已改为“击毁累计保底 + 随机补偿掉落”的混合方案。
- 玩家反馈敌我子弹区分仍不够明确；当前已改为更长更亮的玩家弹体与更大、更亮、带脉冲感的敌弹组合。
- 玩家反馈暂停时敌机仍会移动；当前已修正为战斗节点进入真正的可暂停模式，暂停后敌机、Boss、子弹与背景滚动都会停止。
- 为了提升完成版观感，Boss 出场前预警提示、暂停菜单焦点稳定性，以及射击 / 受击音量舒适度也已做一轮小幅优化。
- 当前 Boss 预警已从普通受击闪屏中独立出来，改为出场前整屏 `DANGEROUS` 警报；高阶主炮也改为按住开火键时稳定常驻并跟随机身移动。

### 当前增强项（2026-04-11）

- 关卡已从双 Sector 继续扩展为五关 + 最终 Boss，整体节奏仍控制在短局街机结构内。
- 时间表继续采用“整波 + 错峰”混合编队，避免均匀铺怪带来的无聊感。
- `spinner` 与 `sentinel` 已加入，用于把后段压力从“更多子弹”改成“更明确的编队与射线要求”。
- 六个 Boss profile 都采用差异轮廓和火力节奏，而不是同一 Boss 单纯换血量。
- Boss 低血量时仍会进入更高压节奏，但避免发展成现代弹幕演出。
- 本轮开始将 Boss 战时长进一步拉长，重点通过提高 Boss 血量而不是纯铺满杂兵来增加压力。
- Boss 战支援小怪已改为更脆的补给目标，保证即便低级武器也能快速击破并争取掉落。
- `repair` 掉落已收紧，整体恢复更依赖操作与节奏控制，而不是连续补血。
- 每一关现在拥有不同的背景主题，展示感和阶段区分更明确。
- 本轮继续将所有关底 Boss 血量再整体抬高到更长局的区间。
- Boss 战小怪从偏随机生成进一步调整为更规整的节奏插入，读图压力更低、预判感更强。
- `shield` 出现频率已继续下调，当前更偏向火力 / 过载补偿，而不是频繁防御兜底。
- 最终 Boss 当前已加入更明确的中段 / 残血阶段强化，后半段火力频率与核心压制感都会明显上升。
- 最终 Boss 击破后的通关收尾已增强，不再是瞬间切场，而是先给出核心摧毁提示与短暂脱离反馈，再进入结果页。
- 菜单中的 `继续` 入口语义已进一步收口，当前明确为阶段继续而非中途断点恢复。
- HUD 状态文本已本地化，不再出现中文界面里混入 `SHIELD / BOOST` 的情况。
- 前五关 Boss 也已加入更明确的阶段变化提示，帮助玩家判断战斗节奏正在抬升。

### 生存模型收紧（2026-04-11）

- 玩家当前为单命 `HULL 3` 结构，不再保留额外 `LIFE` 储备。
- 每次受伤都会优先损失 1 级武器等级；只有跌回基础武器后才会开始扣 `HULL`。
- 为避免节奏因掉级过度停滞，武器保底掉落阈值与随机强化补偿已同步上调。
- 五关 Boss 与最终 Boss 的耐久、支援频率和节奏都做过回收校准。

### 保护盾补偿（2026-04-11）

- 新增 `shield` 拾取，作为单次吸收型增益，优先抵消一次伤害。
- 护盾命中后不会掉血，也不会触发武器降级，用于缓和当前更严格的受伤惩罚。
- 固定关卡节奏点与随机掉落池都已加入护盾，避免恢复节奏完全依赖武器或修复拾取。

### 武器成长扩展（2026-04-11）

- 主武器已进一步扩展为 13 档成长线：
  `PULSE -> DOUBLE -> TRIPLE -> EDGE -> PULSE+ -> DOUBLE+ -> TRIPLE+ -> EDGE+ -> LINE -> LINE+ -> LINE++ -> LINE MAX -> OMEGA`
- 1-4 级负责建立传统扩散成长。
- 5-8 级延续既有模式，但通过加粗弹体与提高压制感体现升级价值。
- 9 级后切换为从机身前沿一路贯穿到屏幕右侧的直线主炮，并在后续升级中持续加粗。
- 固定武器强化节点已重新收紧，避免升级节奏过快。
- 保底武器掉落阈值与随机掉落概率都重新下调，用于适配当前更稳的难度曲线。

### 风险 / 观察项

- 目前缺少覆盖五关 + 最终 Boss 的完整人工试玩记录，手感与难度仍不能只靠自动运行完全确认。
- 9 级以上主炮虽然已经形成明确前方压制，但视觉表现仍可继续增强。
- 目前仍需人工确认 `LINE` 到 `OMEGA` 的束宽成长是否正好落在“压制感足够，但不遮挡阅读”的区间。
- 目前仍需人工确认最终 Boss 的中后段强度是否已经达到“显著高于前五关 Boss，但仍然可学会并通关”的区间。
- 程序化音乐和音效已经可用，但仍偏极简，更适合作为复刻项目完成版而非商业音频方案。
- Headless 强制退出时会看到对象清理警告，需要在本地编辑器实跑确认是否只属于测试退出路径。

## 建议的人工试玩脚本

1. 从主菜单进入游戏，确认最高分和最近一局信息显示正常。
2. 体验至少一整局，记录 Sector 1 到 Sector 5 以及最终 Boss 的难度变化是否合理。
3. 在战斗中触发暂停、继续、重开、回主菜单各至少一次。
4. 确认受击、强化、Boss 出场和通关提示是否足够清楚。
5. 完成或失败后观察结果页、最高分更新和返回路径。

## 当前结论

项目已经达到“可继续维护或展示的复刻项目增强版”状态；当前最需要的不是再扩系统，而是以人工试玩为主继续微调五关与最终 Boss 难度，以及高阶贯穿主炮的手感表现。
