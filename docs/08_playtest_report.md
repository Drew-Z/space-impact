# Playtest Report

## 版本范围

- 版本目标：阶段 4 收口版
- 引擎：Godot 4.6.1
- 测试重点：完整流程、战斗稳定性、HUD 可读性、暂停与结算、音画反馈

## 已完成测试

### 自动化 / 运行验证

- 主菜单场景可启动。
- 战斗场景可启动。
- 结算场景可启动。
- 战斗场景可在固定帧率下连续运行约 4200 帧。
- 未发现新的 GDScript 解析错误或运行时报错。

### 结构与流程检查

- 菜单 -> 战斗 -> 结算主流程已连通。
- 暂停层已存在继续 / 重开 / 返回主菜单入口。
- 最高分与总局数已接入持久化逻辑。
- 双 Sector 流程、双 Boss、三类强化仍然保持可玩闭环。

## 当前体验判断

### 优点

- 项目已经具备“第一次打开就能理解怎么开始、怎么打一局、怎么结束”的完整流程。
- 画面、HUD、暂停页、结果页的风格基本统一。
- 战斗节奏比阶段 1 和阶段 2 更稳，反馈更明确。
- 程序化音效让发射、受击、拾取、Boss 警报和结算都有最低限度的听觉区分。

### 风险 / 观察项

- 目前缺少真实人工试玩记录，手感与难度仍不能只靠自动运行完全确认。
- 程序化音乐和音效已经可用，但仍偏极简，适合原型完成版，不等于最终商业品质音频。
- Headless 强制退出时会看到对象清理警告，需要在本地编辑器实跑确认是否只属于测试退出路径。

## 建议的人工试玩脚本

1. 从主菜单进入游戏，确认最高分和最近一局信息显示正常。
2. 体验至少一整局，记录 Sector 1 与 Sector 2 的难度变化是否合理。
3. 在战斗中触发暂停、继续、重开、回主菜单各至少一次。
4. 确认受击、强化、Boss 出场和通关提示是否足够清楚。
5. 完成或失败后观察结果页、最高分更新和返回路径。

## 当前结论

项目已经达到“可继续维护或展示的复刻项目”状态，但要称为完全收口，仍建议补一轮人工试玩确认与小幅平衡微调。
