# 美术与音频风格

## 视觉方向

- 以 Nokia 3310 单色屏气质为核心。
- 强调清晰轮廓、低细节、高识别度。
- 背景、敌人、子弹、拾取物在亮度和形状上要容易区分。

## 画面原则

- 避免现代炫光、粒子堆叠和高饱和视觉。
- 优先建立“老式设备显示感”，而不是复杂装饰。
- 屏幕空间要留白，保证战斗阅读性。

当前实现补充：

- 战斗背景使用单色底、扫描线、弱栅格和边框来强化设备感。
- UI 面板、菜单、暂停页、结算页统一使用深绿底 + 亮绿描边。
- 反馈动画保持几何化，不使用现代高饱和粒子。

## 占位资源策略

- 阶段 1 可以使用简单几何形或极简像素图。
- 占位资源也要遵守原作气质，不用无关的现代科幻素材。
- 所有占位资源后续都应可替换，不把实现耦合到临时资源上。

## 动画与反馈

- 爆炸、命中、拾取反馈保持短促、清晰、直接。
- Boss 反馈应更明显，但不演出过长。
- HUD 动效尽量克制，信息优先于装饰。

当前实现补充：

- 受击、拾取、敌人击破、Boss 击破统一使用轻量环形 / 十字反馈。
- Boss 与敌人在受击时会短暂闪白，提升命中可读性。
- 阶段开始、Boss 警报、关卡清空使用 HUD 中心提示，不额外叠复杂演出。

## 音频方向

- 音效优先于背景音乐。
- 受击、发射、拾取、爆炸和 Boss 事件需要明确区分。
- 如需音乐，应以简短、节制、复古气质为主。

当前实现补充：

- 使用程序化方波 / 三角波 / 混合噪声生成基础音效。
- 菜单、战斗、暂停、胜利、失败使用不同的极简循环旋律。
- 音频设计目标是“增强节奏与反馈”，而不是现代丰满配乐。

## 允许的现代化微调

- 可使用更清晰的音量层次改善可读性。
- 可对单色方案做轻微屏幕质感模拟，但不应喧宾夺主。
